メールサーバーのことについて調べてみました。
あってるかわかりませんが、
```
# 従来
Rails(ActionMailer) → SMTP(Railsで設定）      →  相手のメールサーバー → ユーザーのメールボックス

# resendを使用する場合
Rails(ActionMailer) → Resend(SMTPなど内部処理) →  相手のメールサーバー → ユーザーのメールボックス
CloudflareでDNS管理としてResendが設定しているSPF/DKIM/DMARCの問い合わせに応答している

Railsはメールを送信する指示出し
 
SMTP(Simple Mail Transfer Protoco)
つまり、メールを送信するためのプロトコル（通信ルール）
従来はRailsでSMTPを設定する必要がある。
ResendがSMTPなどを内部処理してくれる。

Resendはメールを送信するメールサーバー
一緒に、SPF/DKIM/DMARCを発行してくれる（使うか使わないかはCloudeflareに書き込むかどうか次第）

CloudflareはDNS(Domain Name System)サーバーを提供しているサービス。
DNSとは「google.com」のようなドメイン名を、「192.168.x.x」のようなIPアドレスに自動変換する仕組み
それと一緒に、Resendで発行しているSPF/DKIM/DMARCの設定をCloudflareに書き込むと有効になる。

つまり、RailsからResendを経由して相手のメールサーバーに「〇〇@sokonote.com」と届いた時、
相手のメールサーバーが「sokonote.com」のDNS、つまりCloudeflareに問い合わせる。
そこでSPF/DKIM/DMARCを確認して、相手のメールサーバーが最終的にユーザーに「届ける」「迷惑フォルダに入れる」「拒否する」を決定する。

SPF(Sender Policy Framework)
つまり、メールの送信元が正当か（なりすましではないか）確認するための仕組み
DKIM(DomainKeys Identified Mail)電子メールにおける送信ドメイン認証技術の一つ
つまり、電子メールの送信元を確実に認証し、なりすましやメール改ざんを防ぐためのセキュリティ技術
DMARC(Domain-based Message Authentication, Reporting and Conformance)（ディーマーク）
つまり、SPF（正規のサーバーから送られているか？）とDKIM（メールが改ざんされていないか？）を見て
DMARCが「届ける」「迷惑フォルダに入れる」「拒否する」に分ける
```
