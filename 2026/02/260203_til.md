docker compose downしたら、データが全消去されていることに気づいた。
原因はcompose.ymlのvolumes:
誤：postgresql_data:/var/lib/postgresql
正：postgresql_data:/var/lib/postgresql/data
これが原因で保存されてない。
書き直してdocker compose down -vでvolumeも取り壊して、docker compose upしたら
webサーバーが起動しなくなった。
原因は-vでvolumeを全て消してしまったことでnode_modulesなどもまとめて消してしまったから。
とりあえず
docker compose run --rm web yarn install
で入れ直してdocker compose upで正常に動いた。
時間が秒でなくなったよ😭
